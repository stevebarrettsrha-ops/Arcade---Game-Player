==============================================================
  ARCADE — host-side emulators (STREAMING mode)
==============================================================

WHAT THIS IS
  Drop native emulators here to run them ON THIS HOST (e.g. a Steam Deck or
  mini-PC) and STREAM them into ARCADE on your TVs/phones. The host does all
  the work with full power (real GPU + JIT), so you get the heavy / "impossible"
  systems a browser can't do: PS2 (PCSX2), GameCube/Wii (Dolphin), PS3 (RPCS3),
  and flawless N64/PSP — all shown in an iframe on the client devices, driven by
  the ARCADE phone controller.

  This is OPT-IN and experimental. It only switches on when you start the server
  with streaming enabled:
      Linux/Deck :  ARCADE_STREAM=1 node server.js
  Normal browser mode (the rest of ARCADE) is completely unaffected otherwise.

  Full setup (ffmpeg + xdotool + Steam Deck notes): see SETUP-STREAMING.txt
  in the ARCADE folder.

HOW TO ADD AN EMULATOR
  Make ONE folder per emulator in here, each with an "emulator.json" manifest:

      emulators/
        pcsx2/
          emulator.json
          icon.png            (optional cover art)

  emulator.json fields:
      name      shown in ARCADE (e.g. "PCSX2 (PS2)")
      cmd       how to launch it on the host. Use {rom} where the game file
                goes (optional). Examples:
                  "flatpak run net.pcsx2.PCSX2 {rom}"
                  "retroarch -L /path/to/core.so {rom}"
                  "dolphin-emu -b -e {rom}"
                  "/home/deck/Applications/rpcs3.AppImage {rom}"
      capture   how ARCADE grabs the picture:  "x11" (desktop) | "kms" | "test"
      window    (optional) the emulator window's title, so controller input is
                focused on it (e.g. "PCSX2", "Dolphin", "RetroArch")
      needsRom  true if cmd uses {rom}, false for emulators you point at a game
                from their own menu

  EXAMPLE (also see the example-retroarch/ folder):
      {
        "name":    "PCSX2 (PS2)",
        "cmd":     "flatpak run net.pcsx2.PCSX2 {rom}",
        "capture": "x11",
        "window":  "PCSX2",
        "needsRom": true
      }

NOTES
  - "test" capture streams a test pattern (no emulator) so you can confirm the
    video pipeline works before wiring a real emulator.
  - Controller input is sent as keyboard presses (RetroArch's default layout) to
    the focused emulator window. Analog sticks need a virtual gamepad (uinput) —
    that's a later step; digital buttons work now.
  - You supply your own legally-owned emulators and games. Nothing is bundled.
