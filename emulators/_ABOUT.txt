==============================================================
  ARCADE — host-side emulators (STREAMING mode)
==============================================================

WHAT THIS IS
  Drop native emulators here to run them ON THIS HOST (a gaming PC, laptop,
  Steam Deck or mini-PC) and STREAM them into ARCADE on your TVs/phones. The
  host does all the work with full power (real GPU + JIT), so you get the
  heavy / "impossible" systems a browser can't do: PS2 (PCSX2), original
  XBOX (xemu), GameCube/Wii (Dolphin), PS3 (RPCS3), and full-speed N64/PSP —
  shown on the client devices and driven by the ARCADE phone pad.

  This is OPT-IN. It only switches on when you start the server with
  streaming enabled:
      Windows :  start-windows-stream.bat
      macOS   :  start-mac-stream.command
      Linux   :  ./start-linux-stream.sh
  Normal browser mode (the rest of ARCADE) is completely unaffected otherwise.

  Full setup (ffmpeg + per-OS notes): see SETUP-STREAMING.txt in the ARCADE
  folder. Ready-made manifests to copy: see the _examples/ folder in here.

ALREADY SET UP FOR WINDOWS
  Six emulators come pre-configured in here: project64/ (N64), pcsx2/ (PS2),
  ppsspp/ (PSP), xemu/ (XBOX), freej2me/ and kemulator/ (Java). Run
  download-emulators-windows.bat once to fetch the portable ones; the
  installer-based ones (PCSX2/PPSSPP/Project64) you install normally — their
  default paths are pre-wired. Each folder has a _SETUP.txt with specifics.

WRONG PATHS? AUTO-FIX THEM
  The manifests ship with the DEFAULT install paths. If you installed an
  emulator elsewhere, or dropped a portable build straight into its
  emulators/<name>/ folder, the launch silently fails ("button does
  nothing"). Run the path fixer once to scan for where each emulator
  really lives and rewrite the "cmd" paths to match:
      Windows :  fix-emulator-paths-windows.bat   (or: node fix-emulator-paths.js)
      macOS   :  fix-emulator-paths-mac.command
      Linux   :  ./fix-emulator-paths-linux.sh
  Add --dry-run to preview without changing anything. It also creates each
  emulator's games folder and reports how many games it can see.

HOW TO ADD AN EMULATOR
  Make ONE folder per emulator in here, each with an "emulator.json" manifest:

      emulators/
        pcsx2/
          emulator.json
          icon.png            (optional cover art)

  emulator.json fields:
      name      shown in ARCADE (e.g. "PCSX2 (PS2)")
      cmd       how to launch it on the host. Use {rom} where the game file
                goes (optional). QUOTE paths that contain spaces. Examples:
                  "\"C:\\Program Files\\PCSX2\\pcsx2-qt.exe\" -fullscreen {rom}"
                  "flatpak run net.pcsx2.PCSX2 -fullscreen {rom}"
                  "retroarch -f -L mupen64plus_next {rom}"
                  "xemu -full-screen -dvd_path {rom}"
      capture   how ARCADE grabs the picture:
                  "auto"   – whole screen, right grabber per OS (Windows:
                             gdigrab, macOS: avfoundation, Linux: x11). Best
                             when the emulator runs FULLSCREEN (e.g. PCSX2,
                             PPSSPP, xemu all pass a -fullscreen flag) so the
                             screen IS the game.
                  "window" – Windows only: grab ONLY the emulator's window (by
                             its "window" title, below), not the whole desktop.
                             Use this for emulators that run in a WINDOW (e.g.
                             Project64, the Java handsets) so you stream just
                             the game — not your desktop (which would otherwise
                             mirror the browser showing the stream). On macOS/
                             Linux it falls back to a full-screen grab.
                             Note: if the game area shows up BLACK, the
                             emulator is rendering with a backend gdigrab can't
                             read — run it fullscreen and use "auto" instead,
                             or switch its video plugin/backend.
                Or force a specific grabber: x11 | gdigrab | avfoundation |
                kms | test ("test" = a test pattern, no game).
      window    (optional) the emulator window's title. Used to focus
                controller input on Linux/X11 AND, with capture:"window", to
                pick which window to stream. A substring is fine — ARCADE
                matches the live title (e.g. "Project64" matches
                "Mario 64 - Project64"). Examples: "PCSX2", "Project64".
      needsRom  true if cmd uses {rom}; false for emulators you point at a
                game from their own menu
      system    (optional) short label shown on the cards, e.g. "PS2", "XBOX"
      roms      (optional) a folder INSIDE the ARCADE folder whose files are
                listed as launchable games, e.g. "games/ps2". Those games
                then appear in the Host-emulators picker AND in the phone
                pad's Games drawer.
      exts      (optional) which file types in that folder count as games,
                e.g. [".iso", ".chd"] — leave out to list every file.

NOTES
  - Modern devices get an H.264 + AAC stream with audio and video muxed
    together (in sync); old TV browsers fall back to MJPEG + PCM audio.
    It's automatic — ARCADE_STREAM_MODE=auto (default), or force h264/mjpeg.
  - Sound is streamed too: the client shows a "🔊 Tap for sound" button.
    Tell ffmpeg which source to capture with ARCADE_STREAM_AUDIO_DEVICE
    (the speaker monitor/loopback, not a mic) — see SETUP-STREAMING.txt,
    section 3b. ARCADE_STREAM_AUDIO=none turns it off.
  - Controller input is sent as keyboard presses (RetroArch's default
    layout: arrows, Z/X/A/S, Q/W/E/T, Enter, Shift; F2 save, F4 load,
    Space fast-forward). Bind those keys inside the emulator once and the
    phone pad drives it. macOS hosts: the D-pad arrives as I/K/J/L
    (AppleScript can't hold arrow keys) — bind those instead.
  - The phone's analog stick arrives as 8-way digital presses. True analog
    needs a virtual gamepad (uinput) — that's a later step.
  - Folders starting with "_" or "." are ignored (that's why _examples/
    doesn't show up in ARCADE).
  - You supply your own legally-owned emulators, BIOS files and games.
    Nothing is bundled.
